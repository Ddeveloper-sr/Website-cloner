How this works: everything runs in your browser tab — there's no backend. Cross-site fetches are routed through a public CORS proxy since browsers block raw cross-origin reads; proxies can be slow, rate-limited, or temporarily down — use the Test button to check a proxy is alive before starting a full crawl. If the tool detects rate-limiting mid-crawl it automatically slows down for the rest of the run.

Limits: up to 60 assets and 80MB total per clone, to avoid generating a zip too large for a phone browser to handle in memory. Individual files over 8MB are skipped.

What to expect: static HTML, CSS (including animations), images and fonts come through reasonably well on simple sites. Sites built with React/Vue/Next.js often render mostly-empty HTML in the raw response (their content is built by JavaScript after load), so a browser-fetch clone — without a real browser engine running the JS — will look incomplete for those. Login-gated content, anti-bot protection, and strict CSP/CORS policies will also block this.
